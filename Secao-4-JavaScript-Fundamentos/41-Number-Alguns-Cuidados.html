Number: Alguns cuidados

Nessa aula vimos que qualquer número divido por 0 retorna o tipo Infinity. Por exemplo 7 / 0… vai retornar na tela do console o tipo Infinity. Também vimos a divisão de um número ‘10’ por 2 resultando em 5, como já vimos nas aulas anteriores qualquer caractere / símbolo que esteja dentro de aspas simples ou duplas é considerado um texto / string, porém como JavaScript é fracamente tipado ele consegue fazer essa operação e nos retornar o resultado.

Devemos tomar cuidado, para JavaScript executar uma operação entre tipos distintos, ele precisa reconhecer o valor que está entre as aspas, caso contrário o retorno será NaN. Exemplo ‘Show!’ * 2.

Outra operação que o professor Leonardo comentou na aula e a operação entre 0.1 + 0.7, poderíamos esperar o resultado 0.8, porém o JavaScript retorna 0.79, segundo o professor isso ocorre por conta de uma especificação que a maioria das linguagens adotam, que não é 100% precisa, porém é muito mais rápida na sua execução e retornando a resposta em um tempo muito mais rápido.

Tentando converter um número diretamente para String vai dar erro, no exemplo da aula vimos 10.toString() ira dar erro, porém se colocar esse mesmo valor literal entre parênteses a linguagem ira executar a sentença sem nenhum problema.
